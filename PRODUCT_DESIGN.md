# 拾名产品设计

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 拾名（Good Name, Good Life） |
| 文档版本 | 2.1 |
| 更新日期 | 2026-06-21 |
| 当前状态 | 可运行、已测试、持续迭代 |
| 产品形态 | Python 本地服务 + 响应式单页 Web 应用 |

## 1. 产品定位

“拾名”是以中文姓名为核心的传统文化辅助工具。产品将四柱排盘、五行规则、姓名音形义、姓名卦象和组合稀有度转化为可以核对的结构化信息，帮助用户生成、筛选和比较名字。

产品不输出不可解释的“命运结论”，也不把估算结果包装成官方数据。

### 1.1 核心原则

1. 计算透明：每项结果说明规则版本、数据来源和计算依据。
2. 结果一致：评分页与起名页共用唯一评分函数，相同参数得到相同分数。
3. 不造数据：没有合法人口接口时只展示风险等级，不生成虚构人数。
4. 偏好与评分分离：性别、气质和优先用字影响候选范围，不直接改变姓名质量分。
5. 隐私优先：查询历史默认关闭，用户明确同意后才写入 SQLite。
6. 外部数据可追溯：在线字典、姓名语料和人口接口必须返回或配置来源信息。

### 1.2 非目标

- 不提供医疗、法律、投资、婚恋或人生决策建议。
- 不声称命理、五行或姓名卦象具有科学预测能力。
- 不抓取需要登录、验证码或禁止自动访问的公安与第三方网页。
- 不在缺少授权数据时展示“全国同名人数”。

## 2. 目标用户与场景

### 2.1 目标用户

- 为新生儿筛选名字的父母及家庭成员。
- 希望理解现有姓名音形义和五行结构的用户。
- 需要在多组候选名字之间反复比较的用户。
- 关注名字常见程度、但接受风险等级而非虚构人数的用户。

### 2.2 典型任务

1. 输入现有姓名和生辰，查看统一评分及五行分析。
2. 输入姓氏和生辰，获得十组动态候选名。
3. 指定一个优先汉字，在规则允许时尽量纳入候选。
4. 在起名和评分页面之间一键往返，无需重复填写参数。
5. 收藏多个候选，集中比较并导出报告。
6. 输入完整姓名，评估重名风险或调用已授权人口接口。

## 3. 信息架构

```text
首页
├── 姓名评分
│   └── 一键携带参数进入智能起名
├── 智能起名
│   ├── 八字与五行分析
│   ├── 十组候选名字
│   ├── 一键携带参数进入姓名评分
│   └── 收藏、对比、导出和打印
├── AI 姓名顾问
│   ├── 独立参数表单
│   ├── 语义与文化意象
│   └── 谐音及歧义风险
└── 重名风险
    ├── 默认：透明规则风险评估
    └── 已配置授权接口：真实人数及来源
```

独立“卦象解析”已移除。卦象只保留为姓名评分的一个解释维度，避免产品偏离姓名主题。

## 4. 功能规格

### 4.1 姓名评分

输入：

- 完整中文姓名，2–6 个字符。
- 公历出生日期，范围为 1900 年至当前日期。
- 十二时辰。
- 性别。
- 五行规则版本。
- 评分版本。

处理流程：

1. 使用节气万年历计算年、月、日、时四柱。
2. 统计八个干支本气的木、火、土、金、水数量。
3. 查询姓名各字的笔画、五行、释义和来源。
4. 使用 `pypinyin` 获取声调信息。
5. 计算音韵、字形、寓意、五行和卦象五项得分。
6. 按选定评分版本加权生成总分。

输出：

- 总分与评价等级。
- 五个维度的分数。
- 四柱、农历、日主、月令和五行计数。
- 五行规则、评分版本和逐字来源。
- 姓名上下卦及综合说明。
- “按此生辰智能起名”入口。

### 4.2 智能起名

输入：

- 姓氏，支持单姓和已收录复姓。
- 公历出生日期与十二时辰。
- 性别：男孩、女孩或不限定。
- 期望气质：清雅从容、明朗大气或博学睿智。
- 优先用字，可为空或填写一个中文汉字。
- 五行规则版本与评分版本。

候选生成：

1. 按五行规则确定必补元素、推荐元素和相生辅助元素。
2. 从按五行分类的动态用字库组合双字名。
3. 排除重复字、与姓氏重复的字和明显不符合性别的用字。
4. 优先用字只影响候选优先级，不进入姓名评分。
5. 气质风格只影响候选排序，不进入姓名评分。
6. 若配置授权姓名语料，使用组合频次改进自然度，并过滤歧义词。
7. 对最多 120 组候选调用统一评分函数，返回十组结果。

输出：

- 完整八字和五行分析。
- 当前规则及语料来源。
- 十组姓名、分数、寓意、用字五行和姓名卦象。
- 语料歧义提示。
- “查看评分”和“收藏”操作。

性能目标：正常本地请求在 2 秒内完成；当前核心函数测试通常低于 1 秒。

### 4.3 参数双向传递

姓名评分进入智能起名时自动传递：

- 姓氏。
- 出生日期和时辰。
- 性别。
- 五行规则版本。
- 评分版本。

候选名进入姓名评分时自动传递：

- 完整姓名。
- 出生日期和时辰。
- 性别。
- 五行规则版本。
- 评分版本。

目标：用户只填写一次基础资料，即可往返比较多个候选名字。

### 4.4 收藏、对比和导出

- 收藏数据保存在浏览器 `localStorage`。
- 相同姓名、生辰和规则版本组成唯一收藏标识，避免重复收藏。
- 对比区展示姓名、分数、寓意、五行、生辰和时辰。
- 用户可以移除单个候选。
- 支持导出 UTF-8 JSON 报告。
- 支持浏览器打印，可另存为 PDF。

收藏属于用户主动保存的浏览器数据，与服务器“查询历史”开关相互独立。

### 4.5 重名风险与真实人口接口

默认模式根据以下特征输出 0–100 风险分、等级、置信度和逐条依据：

- 姓氏常见度分档。
- 高频起名字命中数量。
- 单字名、双字名或多字名结构。
- 叠字。
- 常见完整姓名命中情况。

默认模式不输出估算人数。

配置授权人口 API 后，页面改为展示：

- 真实同名总人数。
- 男女人数（接口提供时）。
- 数据来源。
- 查询时间。

## 5. 排盘与规则版本

### 5.1 排盘基线

- 数据源：`lunar-python 1.4.8`。
- 时间标准：北京时间。
- 日界配置：Sect 2。
- 输入粒度：十二时辰。
- 当前不计算出生地经度和真太阳时。

### 5.2 五行规则版本

#### 缺失补足 `missing_v1`

- 统计四柱八个干支本气。
- 计数为 0 的元素标记为缺失。
- 双字名最多把前两个缺失元素作为硬约束。
- 没有缺失时，选择计数最少的最多两个元素。

#### 月令平衡 `seasonal_v2`

- 在本气计数基础上，对月令五行增加权重 2。
- 选择加权后计数最少的最多两个元素。
- 不把“原始计数为 0”自动设为硬约束。

以上均为产品内明确、可复算的规则，不代表所有传统命理流派的统一喜用神结论。

### 5.3 评分版本

| 指标 | 综合均衡 `balanced_v1` | 五行侧重 `element_v2` |
| --- | ---: | ---: |
| 音韵 | 18% | 15% |
| 字形 | 14% | 12% |
| 寓意 | 18% | 15% |
| 五行 | 35% | 43% |
| 卦象 | 15% | 15% |

风格、优先用字和生成随机种子禁止进入评分。

### 5.4 单项指标

- 音韵：拼音声调的相邻变化与组合丰富度。
- 字形：姓名各字笔画差和相邻笔画跨度。
- 寓意：给定名字在释义字库中的覆盖率。
- 五行：给定名字对必补或推荐元素的覆盖情况。
- 卦象：姓氏笔画取上卦、名字笔画取下卦后的五行匹配。

## 6. 数据设计

### 6.1 内置汉字数据

每个汉字包含：

- 简体笔画数。
- 姓名学五行。
- 简短释义。
- 来源标签。

未收录字不得通过 Unicode 编码、哈希或随机数猜测属性。

### 6.2 授权汉字 API

环境变量：

```text
SHIMING_CHAR_API_URL
SHIMING_CHAR_API_KEY
```

请求：

```json
{ "char": "珩" }
```

响应：

```json
{
  "strokes": 10,
  "element": "金",
  "meaning": "美玉端方",
  "source": "字典名称及版本"
}
```

接口返回字段不完整、元素非法、超时或无来源时，系统明确报错，不使用猜测值。

### 6.3 授权姓名语料

环境变量：`SHIMING_NAME_CORPUS`。

```json
{
  "source": "数据集名称、版本与授权说明",
  "bigrams": { "安宁": 120, "知远": 85 },
  "ambiguous_terms": ["需规避的歧义词"]
}
```

- `bigrams` 用于候选自然度排序，不进入姓名评分。
- `ambiguous_terms` 用于显示歧义提示。
- 未配置语料时明确显示“未配置授权姓名语料”。

### 6.4 授权人口 API

环境变量：

```text
SHIMING_POPULATION_API_URL
SHIMING_POPULATION_API_KEY
```

请求：

```json
{ "name": "张伟" }
```

响应：

```json
{
  "total": 12345,
  "male": 9000,
  "female": 3345,
  "source": "授权人口数据服务名称"
}
```

`total` 和 `source` 必填；未配置接口时继续使用透明风险模型。

## 7. 隐私与存储

### 7.1 服务器查询历史

- 默认关闭。
- 页面开关开启后，请求携带 `X-Store-History: true`。
- 仅在收到该请求头时创建或写入 `shiming.db`。
- 数据库文件被 Git 忽略。
- SQLite 连接每次显式关闭，避免 Windows 文件锁。

### 7.2 浏览器收藏

- 收藏由用户主动触发。
- 数据保存在当前浏览器 `localStorage`。
- 当前版本支持单条删除，不上传服务器。

### 7.3 凭据

- API Key 仅通过环境变量读取。
- Key、数据库、缓存、日志和本地认证工具不得提交 Git。
- API 响应中的来源信息可以展示，凭据不得返回前端。

## 8. API

| 方法 | 路径 | 功能 |
| --- | --- | --- |
| GET | `/api/health` | 服务健康检查 |
| POST | `/api/score` | 姓名评分 |
| POST | `/api/names` | 智能起名 |
| POST | `/api/population` | 重名风险或授权人口查询 |

统一成功响应：

```json
{ "ok": true, "data": {} }
```

统一失败响应：

```json
{ "ok": false, "error": "清晰的中文错误信息" }
```

请求体最大 16 KB。业务校验错误返回 HTTP 400；不存在的接口返回 404；未处理异常返回 500。

## 9. 交互与视觉

- 东方纸张、印章和水墨视觉语言。
- 桌面端双栏输入与结果布局。
- 移动端单栏响应式布局。
- 请求期间按钮禁用并显示“正在计算”。
- 评分和起名页面保留参数，支持自动提交。
- 打印模式隐藏导航、输入表单和操作按钮，突出结果与收藏。

## 10. 非功能要求

- 支持 Python 3.10 及以上版本。
- 本地服务默认监听 `127.0.0.1:8000`。
- 智能起名目标响应时间小于 2 秒。
- 相同姓名、生辰和规则版本的分数必须一致。
- 所有用户输入进行长度、日期和汉字范围校验。
- 外部 API 设置 8–10 秒超时。
- 不将数据库、缓存、日志、密钥和本地工具提交 Git。

## 11. 验收标准

### 11.1 自动测试

- 起名页候选分数与评分页结果一致。
- 两个五行规则版本和两个评分版本均可使用。
- 查询历史关闭时不创建数据库，开启后正常写入。
- 未配置人口 API 时返回风险模式且不包含人数。

当前回归测试：4 项全部通过。

### 11.2 人工验收

- 姓名评分与智能起名可双向跳转且参数完整。
- 优先用字为空时保持默认逻辑，填写后优先出现在候选中。
- 收藏后刷新页面仍保留候选。
- JSON 导出文件可解析，打印报告排版完整。
- 关闭历史开关后，API 请求不写 SQLite。
- 手机和桌面页面均无横向溢出。

## 12. 已知限制

- 内置字库覆盖有限，生僻字需要授权在线字典。
- 五行归类、姓名卦象和喜用判断存在流派差异。
- 十二时辰无法处理边界分钟及真太阳时差异。
- 未配置授权姓名语料时，只能依靠内置字义和规则组合，不能代表真实姓名语言分布。
- 风险模型没有公安人口样本，默认只能输出启发式风险等级。
- 收藏数据当前没有“一键全部清空”操作。
