# 拾名产品设计

## 1. 产品定位

“拾名”是一个以中文姓名为核心的传统文化辅助工具。产品目标不是输出不可解释的“命运结论”，而是把四柱排盘、五行规则、姓名音形义和组合风险转化为可以核对的结构化信息，帮助用户筛选和比较名字。

核心原则：

1. 计算过程透明，不使用随机分数冒充分析结果。
2. 同一输入得到一致评分，评分页与起名页共用同一算法。
3. 真实数据不可得时明确说明边界，不虚构人口人数或官方结论。
4. 用户偏好只影响推荐范围，不改变姓名自身评分。

## 2. 目标用户

- 为新生儿筛选名字的父母及家庭成员。
- 希望理解现有姓名音形义和五行结构的用户。
- 需要比较多组候选名字并关注重名风险的用户。

## 3. 核心流程

### 3.1 姓名评分

用户输入完整姓名、出生日期、十二时辰和性别。系统依次完成：

1. 按节气排出年、月、日、时四柱。
2. 统计八个干支本气的木、火、土、金、水数量。
3. 解析姓名用字的笔画、五行、拼音声调和内置释义。
4. 计算音韵、字形、寓意、五行和卦象五项得分。
5. 输出总分、明细、五行分析和可核对的计算依据。
6. 用户可一键携带原参数进入智能起名。

### 3.2 智能起名

用户输入姓氏、出生日期、十二时辰、性别、期望气质，可选填一个优先用字。

1. 系统先展示四柱、农历日期、日主、月令和五行数量。
2. 五行存在缺失时，将缺失元素作为候选名硬约束。
3. 五行齐全时，优先考虑计数最少元素及其相生元素。
4. 性别用于排除明显不匹配的用字；气质偏好只影响候选排序。
5. 优先用字在不破坏五行约束和基本质量时尽量采用。
6. 从动态字库组合中精评候选，返回十组名字、分数、寓意、用字五行和姓名卦象。
7. 每个名字可一键携带原参数进入姓名评分，实现往返比较。

### 3.3 重名风险

用户输入完整姓名。系统根据姓氏分档、常见起名字、名字长度、叠字和常见完整姓名表，输出 0–100 风险分、风险等级、模型置信度和逐条判断依据。

该模块不展示“全国同名人数”，因为没有合法人口接口时，任何精确人数都是不可验证的。

## 4. 规则与数据

### 4.1 排盘

- 数据源：`lunar-python 1.4.8`。
- 输入：公历日期和十二时辰。
- 时间标准：北京时间。
- 日界流派：使用库的 Sect 2 配置。
- 当前不处理出生地经度和真太阳时。

### 4.2 五行补益

- 统计对象：四柱的八个干支本气。
- 计数为 0：标记为缺失，起名时必须覆盖；若缺失超过两个元素，双字名最多直接覆盖两个，应在产品中提示限制。
- 无缺失：选择计数最少的最多两个元素作为推荐方向。
- 相生关系：木生火、火生土、土生金、金生水、水生木。

“缺什么补什么”是当前采用的产品规则，不应标注为统一的传统命理定论。

### 4.3 姓名评分

总分计算：

```text
总分 = 音韵×18% + 字形×14% + 寓意×18% + 五行×35% + 卦象×15%
```

- 音韵：依据拼音声调的相邻变化和组合丰富度。
- 字形：依据姓名各字笔画差和相邻笔画跨度。
- 寓意：依据给定名字在释义字库中的覆盖率。
- 五行：依据给定名字是否覆盖缺失或少量元素。
- 卦象：姓氏笔画取上卦，名字笔画取下卦，再计算卦象五行匹配。

气质偏好和生成随机种子不得进入姓名评分。

### 4.4 字库

每个收录汉字包含笔画、五行和简短释义。未收录字必须返回明确错误，不能通过 Unicode 编码、哈希或随机数猜测属性。

后续若接入在线字典，应满足：

- API 具有明确授权和稳定服务条款。
- 保存原始来源、查询时间和规则版本。
- 在线接口失败时不伪造结果。
- 五行字段没有统一标准时，必须展示所采用的流派或换算规则。

## 5. 页面结构

```text
首页
├── 姓名评分
├── 智能起名
└── 重名风险
```

独立“卦象解析”已移除；卦象只作为姓名评分中的一个解释维度，避免偏离姓名产品定位。

## 6. 非功能要求

- 桌面端和移动端响应式布局。
- 智能起名正常请求目标响应时间小于 2 秒。
- 所有用户输入进行长度、日期和汉字范围校验。
- API 错误返回清晰中文信息，页面按钮在请求期间防止重复提交。
- 本地数据库、缓存、安装文件和认证工具不得提交 Git。

## 7. 已知限制

- 字库覆盖范围仍有限，生僻字可能无法评分。
- 五行归类和姓名卦象存在流派差异。
- 十二时辰粒度无法处理时辰边界附近的分钟误差。
- 动态组合保证规则匹配，但尚未使用大规模真实姓名语料进行搭配自然度训练。
- 重名模型没有公安人口样本，只能输出启发式风险等级。

## 8. 后续规划

1. 引入有授权的汉字笔画、释义在线 API，并实现来源标记。
2. 增加用户可选择的五行流派和评分规则版本。
3. 引入合法中文姓名语料，改进搭配自然度和歧义检测。
4. 增加候选收藏、对比表和导出报告。
5. 提供隐私开关，允许完全关闭本地历史记录。
6. 获得合法人口数据授权后，再增加真实同名人数查询。
